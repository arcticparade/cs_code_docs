# Who Should Read What

Use this page when you are not sure where to start.

## If you use the product in the browser

Start with:

- [quickstart.md](quickstart.md)
- [user-guide.md](user-guide.md)
- [workspace-reference.md](workspace-reference.md)

This path is best for people who mainly work in Studio and want to understand the normal product flow.

## If you want to automate from a terminal

Start with:

- [cli-reference.md](cli-reference.md)
- [auth-reference.md](auth-reference.md)
- [security-features.md](security-features.md)

This path is best for developers, operators, and technical users who want to script routing or execution tasks.

## If you want to integrate with the API

Start with:

- [api-reference.md](api-reference.md)
- [api-routing.md](api-routing.md)
- [api-conversations.md](api-conversations.md)

Then continue to:

- [api-operations.md](api-operations.md)
- [api-billing.md](api-billing.md)

This path is best for teams building direct integrations or internal tooling around CS Code Router.

## If you want to understand why routes change

Start with:

- [model-routing.md](model-routing.md)
- [security-features.md](security-features.md)

This path is best for people who want to understand model choice, fallback behavior, memory, and product guardrails.
